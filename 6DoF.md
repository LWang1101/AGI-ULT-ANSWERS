# 🧠 1. 四元数为什么“像复数但维度是 4”

## 1.1 复数的结构

复数 **z = a + bi** 有两个维度：

* 实部 a
* 虚部 b，且满足 **i² = -1**

复数乘法之所以能表示二维旋转，是因为：

* 乘法会产生「旋转 + 缩放」效果
* i 的平方为 -1，赋予了旋转性质

---

## 1.2 四元数的结构

四元数是 **复数向 3D 推广后的产物**：

[
q = w + xi + yj + zk
]

含 4 个分量：

* **w**：实部
* **x, y, z**：虚部（类似 3D 向量）

就像复数有 i，
四元数有 **i、j、k** 三个虚单位。

---

# 🔺 2. 四元数“乘法规则特殊”的原因

## 2.1 基本规则

四元数的虚单位满足：

[
i^2 = j^2 = k^2 = ijk = -1
]

以及三条关键的乘法关系：

[
ij = k,\quad ji = -k
]
[
jk = i,\quad kj = -i
]
[
ki = j,\quad ik = -j
]

### ⭐ 重要：四元数乘法是「非交换」的

也就是：

> **pq ≠ qp**

这是它跟向量和复数最大的不一样，也正是它能完美描述 3D 旋转的原因。

---

## 2.2 直觉解释

### 为什么像复数？

复数的 i 是“一个虚方向”，
四元数的 i, j, k 是“3 个互相正交的虚方向”。

### 四元数乘法的本质

四元数乘法**相当于把两个旋转组合到一起**：

* p 代表一个旋转
* q 代表另一个旋转
* p*q 代表“先做 q 再做 p”

顺序不同结果不同 → 所以乘法也不能交换。

---

## 2.3 更直白的比喻

假设你要：

* 先往左转 90°
* 再低头 90°

和：

* 先低头 90°
* 再往左转 90°

最终指向完全不一样。

四元数为了表达这种“旋转顺序影响结果”，乘法就必须是非交换的。

---

# 🧮 3. 四元数乘法的数学公式

给两个四元数：

[
p = (w_1, \vec{v_1})
]
[
q = (w_2, \vec{v_2})
]

其中 (\vec{v}) 是三维向量 (x, y, z)。

乘法结果：

[
p \otimes q =
(w_1 w_2 - \vec{v_1}\cdot\vec{v_2},;
w_1 \vec{v_2} + w_2 \vec{v_1} + \vec{v_1}\times\vec{v_2})
]

你可以看到：

* 用到了 **点积（·）**
* 也用到了 **叉积（×）**

这就是四元数与 3D 向量关系非常紧密的原因。

# 🧪 4. 极简伪代码（可直接转为 Python/C++/Unity）

### 数据结构

```python
# 四元数 q = w + xi + yj + zk
struct Quaternion:
    float w, x, y, z
```

---

## 4.1 四元数乘法（关键）

```python
def quat_mul(p, q):
    # p and q are quaternions
    w1, x1, y1, z1 = p.w, p.x, p.y, p.z
    w2, x2, y2, z2 = q.w, q.x, q.y, q.z

    w = w1*w2 - x1*x2 - y1*y2 - z1*z2
    x = w1*x2 + x1*w2 + y1*z2 - z1*y2
    y = w1*y2 - x1*z2 + y1*w2 + z1*x2
    z = w1*z2 + x1*y2 - y1*x2 + z1*w2

    return Quaternion(w, x, y, z)
```

这段代码实现了四元数的核心乘法规则，适用于：

* Unity (C#)
* C++
* Python
* JavaScript
  你只需要调整语法即可。

# 4.2 单位四元数正规化（必须做）

因为数值误差会累积：

```python
def normalize(q):
    mag = sqrt(q.w*q.w + q.x*q.x + q.y*q.y + q.z*q.z)
    return Quaternion(q.w/mag, q.x/mag, q.y/mag, q.z/mag)
```

# 4.3 用四元数旋转一个向量（重要）

旋转 v 的公式：

[
v' = q \otimes (0, v) \otimes q^{-1}
]

```python
def rotate_vector(q, v):
    # v is (vx, vy, vz)
    qv = Quaternion(0, v.x, v.y, v.z)
    q_conj = Quaternion(q.w, -q.x, -q.y, -q.z)

    res = quat_mul(quat_mul(q, qv), q_conj)
    return Vector3(res.x, res.y, res.z)
```

> 因此是 3D 旋转最稳定、最优雅的数学表示方式。
